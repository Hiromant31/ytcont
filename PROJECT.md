# YTCont - AI Video Production Pipeline
> Автоматизированная система создания видеороликов: на входе - текстовая идея, на выходе - готовое видео с анимацией, озвучкой и субтитрами

## 🧠 Суть и логика
Полный цикл от идеи до готового видео через 8 последовательных этапов обработки. Веб-интерфейс → оркестрация этапов → локальная/облачная обработка → финальный рендеринг.

**Pipeline пошагово:**
1. Пользователь вводит идею в веб-интерфейс и выбирает настройки
2. Система генерирует сценарий из 3 эпизодов (завязка/кульминация/финал)
3. Сценарий разбивается на сцены (раскадровка)
4. Генерируются референсы персонажей
5. Для каждой сцены создаются изображения
6. Текст каждой сцены озвучивается (TTS)
7. Из аудио извлекаются субтитры (Whisper)
8. Создается манифест сборки
9. Финальное видео рендерится с анимацией и субтитрами

## 🖥 Среда запуска

**Режимы работы:**
1. **Локальный режим** (обычный): Все вычисления на локальной машине
2. **Гибридный режим** (рекомендуемый): Локальный UI + Colab GPU для Whisper и рендеринга

**Операционные системы:**
- Ubuntu 20.04+ (рекомендуется)
- macOS (поддерживается)
- Windows (с ограничениями, не протестировано)

**Требования к железу:**
- **RAM:** Минимум 8GB (рекомендуется 16GB)
- **CPU:** Современный 4+ ядерный процессор
- **GPU:** Не обязательно для локального режима (но ускоряет Whisper)
- **Диск:** 10GB свободного места для временных файлов и кэша

**Особые требования:**
- **FFmpeg** должен быть установлен и доступен в PATH
- **Интернет соединение** для работы с API (Yandex AI, Colab)
- **Python 3.12+** с virtualenv

**Команда запуска:**
```
cd ytcont
source venv/bin/activate  # Активировать virtualenv
python src/main.py        # Запустить веб-сервер
```

Сервер запустится на `http://localhost:8000` с веб-интерфейсом.

**Для гибридного режима:**

# Предварительно установить pyngrok

## 🔄 Полный pipeline (подробно)

**Этап 1: Генерация сценария**
- **Файл:** `src/stage_1_story.py`
- **API:** Yandex AI (gemma-3-27b-it)
- **Вход:** Текстовая идея из `idea.txt` или UI
- **Выход:** Сценарий из 3 эпизодов в `data/story_draft.txt`
- **Время:** 15-30 секунд

**Этап 2: Создание раскадровки**
- **Файл:** `src/stage_2_scenes.py`
- **API:** Yandex AI (gemma-3-27b-it)
- **Вход:** Сценарий из `story_draft.txt`
- **Выход:** Структура сцен в `data/1_base_structure.json`
- **Время:** 10-20 секунд

**Этап 3: Референсы персонажей**
- **Файл:** `src/stage_3_char_ref.py`
- **API:** Yandex Vision (генерация изображений)
- **Вход:** Описания персонажей из раскадровки
- **Выход:** PNG файлы в `outputs/references/`
- **Время:** 30-60 секунд (зависит от количества персонажей)

**Этап 4: Генерация изображений для сцен**
- **Файл:** `src/stage_4_yandex_scenes.py`
- **API:** Yandex Vision (генерация изображений)
- **Вход:** Описания сцен из `1_base_structure.json`
- **Выход:** PNG изображения для каждой сцены (временные файлы)
- **Время:** 2-5 минут (зависит от количества сцен)

**Этап 5: Озвучивание текста (TTS)**
- **Файл:** `src/stage_5_tts_yandex.py`
- **API:** Yandex TTS
- **Вход:** Текст каждой сцены
- **Выход:** MP3 файлы в `outputs/audio/`
- **Время:** 1-3 минуты (зависит от длины текста)

**Этап 6: Извлечение субтитров**
- **Модуль:** `src/stage_5_5_subtitles.py` (локальный) ИЛИ Colab воркер (Whisper)
- **Инструмент:** OpenAI Whisper
- **Вход:** MP3 файлы озвучки
- **Выход:** JSON субтитров в `outputs/subtitles/` (группировка по 3 слова)
- **Время:** 1-2 минуты (локально) или 30-60 секунд (Colab GPU)

**Этап 7: Создание манифеста**
- **Файл:** `src/stage_6_build_manifest.py`
- **Алгоритм:** Сборка конфигурации рендеринга
- **Вход:** Изображения + аудио + субтитры
- **Выход:** `data/3_assembly_manifest.json`
- **Время:** 5-10 секунд

**Этап 8: Рендеринг видео**
- **Модуль:** `src/stage_7_renderer.py` (локальный) ИЛИ Colab воркер (FFmpeg GPU)
- **Инструмент:** FFmpeg + Moviepy
- **Вход:** Манифест из `3_assembly_manifest.json`
- **Выход:** MP4 видео в `outputs/final_videos/`
- **Время:** 2-5 минут (CPU) или 30-60 секунд (GPU в Colab)

**Общее время пайплайна:**
- Локальный режим: 8-15 минут
- Гибридный режим: 3-8 минут (благодаря Colab GPU)

## 🚨 Особенности, костыли, важные нюансы

1. **Анимация сцен:**
   - Используется Moviepy с плавными переходами (zoom_in, pan_left, fade_in)
   - Каждая анимация настраивается отдельно в production map
   - [ВАЖНО] При рендеринге на CPU анимации могут быть медленными

2. **Группировка субтитров:**
   - Автоматическая группировка по 3 слова на строку
   - Для русского языка работает хорошо, для английского может требовать настройки
   - Группировка происходит в stage_5_5_subtitles.py функция `create_subtitles()`

3. **Proxy управление для Colab:**
   - В orchestrator.py есть сложная логика управления дочерними процессами
   - Используется ngrok для туннелирования коллаба
   - [КОСТЫЛЬ] Если Colab падает или меняется URL - нужно перезапустить вручную

4. **Кодирование Yandex API ключей:**
   - В orchestrator.py строки 18-21 жестко закодированы API ключи
   - Должны быть перенесены в .env [ТРЕБУЕТ РЕФАКТОРИНГА]

5. **Настройки в settings.json:**
   ```json
   {
     "stage": 6,                    # Текущий этап пайплайна
     "aspect_ratio": "9:16",        # Формат (вертикальное видео)
     "quality": "240p",            # Разрешение (256x454 для 9:16)
     "codec": "libx264",           # Для GPU: "h264_nvenc"
     "test_mode": true,            # Сокращенные версии для тестов
     "use_colab_whisper": true,    # Использовать Colab для Whisper
     "use_colab_render": true,     # Использовать Colab для рендеринга
     "colab_url": "https://db5d-35-232-85-131.ngrok-free.app"  # Proxy URL
   }
   ```

6. **Альтернативные реализации и дублирование:**
   - В проекте 26 Python файлов, многие из них - альтернативные версии
   - Основная пайплайн использует: stage_1_story.py, stage_2_scenes.py, stage_3_char_ref.py, stage_4_yandex_scenes.py, stage_5_tts_yandex.py, stage_5_5_subtitles.py, stage_6_build_manifest.py, stage_7_renderer.py
   - Альтернативы: *Goog.py (Google Gemini), *GogColab.py (Google Colab), *manim.py (математические анимации)
   - OL* copy.py - старые версии, не используются
   - [ВАЖНО] При модификациях нужно обновлять все соответствующие версии

7. **Промпты для AI:**
   - Хранятся в `prompts/` как простые текстовые файлы
   - Каждый промпт оптимизирован для конкретной задачи
   - Могут требовать регулировки для улучшения качества сценария

8. **Ошибки и обработка:**
   - Если Whisper падает на Colab, попробуйте локальный режим
   - При низкой скорости интернета Colab может отключаться
   - На этапе генерации изображений возможны таймауты Yandex API

9. **Шрифты для субтитров:**
   - Используется DejaVu Sans (лежит в assets/dejavu_sans/)
   - Для изменения шрифта нужно перезаписать путь в stage_7_renderer.py

10. **Захардкоженные значения [УЧЕСТЬ]:**
   - `YANDEX_FOLDER_ID` в orchestrator.py:18
   - URL API Yandex в stage_4_yandex_scenes.py
   - Пути к шрифтам в stage_7_renderer.py
   - Порты для ngrok (8000) в dowork.py
