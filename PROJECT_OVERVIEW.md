# YTCont - AI Video Production Pipeline

## Обзор проекта

Автоматизированная система для создания видеороликов с использованием AI. Проект состоит из веб-интерфейса для управления пайплайном и воркера для обработки на Google Colab с GPU.

## Архитектура

### Основные компоненты

1. **main.py** - FastAPI веб-сервер с UI
   - Веб-интерфейс для управления пайплайном
   - API endpoints: `/start`, `/status`, `/settings`
   - Интеграция с ngrok для публичного доступа

2. **orchestrator.py** - Менеджер производственного пайплайна
   - Управление 8 этапами создания видео
   - Логирование и отслеживание статуса
   - Гибридный режим: локальная/облачная обработка

3. **dowork.py** - Colab GPU воркер
   - FastAPI сервер для удаленной обработки
   - Endpoints: `/whisper` (транскрипция), `/render` (рендеринг)
   - FFmpeg обработка с GPU ускорением

## Пайплайн производства (8 этапов)

1. **Сценарий** - Генерация сценария из идеи
2. **Раскадровка** - Разбивка на сцены
3. **Референсы лиц** - Создание персонажей
4. **Генерация кадров** - Создание изображений для сцен
5. **Озвучка** - TTS через Yandex
6. **Субтитры** - Whisper транскрипция (локально или Colab)
7. **Манифест** - Сборка метаданных
8. **Рендеринг** - Финальная сборка видео (локально или Colab)

## Технологический стек

- **Backend**: FastAPI, Python
- **Обработка видео**: FFmpeg (CPU/GPU)
- **AI**: Whisper (транскрипция), Yandex TTS
- **Инфраструктура**: Google Colab, ngrok
- **Frontend**: Vanilla JS, HTML/CSS

## Режимы работы

### Локальный режим
- Все этапы выполняются на локальной машине
- Требует установленных зависимостей

### Гибридный режим (Colab)
- Субтитры: Whisper на Colab GPU
- Рендеринг: FFmpeg с h264_nvenc на Colab GPU
- Остальные этапы локально

## Структура данных

### Входные данные
- `idea.txt` - Идея для видео
- `settings.json` - Конфигурация пайплайна

### Выходные данные
- `outputs/audio/{episode}/` - Аудиофайлы сцен
- `outputs/subtitles/{episode}.json` - Субтитры по сценам
- `outputs/final_videos/{episode}.mp4` - Готовые видео
- `data/3_assembly_manifest.json` - Манифест сборки

## Конфигурация

### Параметры видео
- **Формат**: 9:16 (Shorts/TikTok) или 16:9 (YouTube)
- **Качество**: 240p - 1080p
- **Кодек**: libx264 (CPU) или h264_nvenc (GPU)

### Параметры Colab
- `use_colab_whisper` - Использовать Colab для субтитров
- `use_colab_render` - Использовать Colab для рендеринга
- `colab_url` - URL ngrok воркера

## Особенности реализации

### Proxy управление
- Автоматическое отключение proxy для Colab запросов
- Восстановление proxy после завершения

### Обработка субтитров
- Группировка слов по 3 для читаемости
- Подсветка текущего слова желтым цветом
- ASS формат для наложения на видео

### Анимация сцен
- zoom_in, zoom_out, pan_left, pan_right
- 24 FPS, динамическое масштабирование
- Upscale 2x перед анимацией для качества

### Отказоустойчивость
- Логирование всех этапов с timestamp
- Обработка ошибок с traceback
- Возможность запуска с любого этапа

## Детальное описание модулей

### main.py - Веб-интерфейс и API

**Назначение**: Главный сервер приложения с веб-интерфейсом для управления пайплайном.

#### Endpoints

**GET /**
- Возвращает HTML интерфейс
- Встроенный CSS/JS (single-page app)
- Дизайн: темная тема, градиенты, адаптивная верстка

**POST /start**
- Запускает пайплайн в фоновом режиме
- Принимает: `StartRequest` (stage, idea, настройки видео, Colab параметры)
- Сохраняет настройки в `settings.json`
- Возвращает: `{"status": "started"}`

**GET /status**
- Возвращает текущее состояние пайплайна
- Формат: `{"stage": int, "logs": [string]}`
- Используется для polling каждые 1.5 сек

**GET /settings**
- Возвращает сохраненные настройки из `settings.json`
- Используется при загрузке страницы

#### UI Features

**Основные элементы**:
- Textarea для ввода идеи/сценария
- Селекторы: формат (9:16/16:9), качество (240p-1080p), кодек
- Toggle для тестового режима (первые 15 сек)
- Секция Colab с переключателями Whisper/Render
- Поле для ngrok URL

**Pipeline визуализация**:
- 8 этапов с иконками и номерами
- Подсветка активного этапа
- Кнопка запуска для каждого этапа
- Badges для Colab-этапов (субтитры, рендер)

**Логи**:
- Реалтайм отображение с автоскроллом
- Цветовая кодировка:
  - Зеленый: успех (✅, 🎉)
  - Красный: ошибки (❌, ⚠️)
  - Белый жирный: заголовки этапов (━━━)
  - Голубой: Colab операции (☁️, 📤, 📥)

**Статус индикатор**:
- Серый: ожидание
- Оранжевый (пульсация): выполнение
- Зеленый: завершено
- Красный: ошибка

#### Технические детали

**Ngrok интеграция**:
```python
if os.getenv("NGROK_AUTH_TOKEN"):
    public_url = ngrok.connect(8000).public_url
```
- Автоматический публичный URL если задан токен
- Порт: 8000 (локальный), 127.0.0.1

**Background tasks**:
- Использует FastAPI BackgroundTasks
- Пайплайн запускается асинхронно
- Не блокирует API ответы

**Pydantic модель StartRequest**:
- Валидация входных данных
- Значения по умолчанию для всех полей
- Сериализация в JSON для settings.json

### orchestrator.py - Менеджер пайплайна

**Назначение**: Центральный оркестратор всех этапов производства видео. Управляет последовательным выполнением, логированием и гибридным режимом (локально/Colab).

#### Класс VideoProductionManager

**Состояние**:
- `status` - текущий статус ("Ожидание", "Выполнение: {этап}", "Завершено", "Ошибка")
- `current_stage_idx` - номер активного этапа (0-7)
- `logs` - массив логов с timestamp
- `is_running` - флаг выполнения
- `settings_path` - путь к settings.json

**Этапы пайплайна** (self.stages):
1. Сценарий → `run_stage_1()`
2. Раскадровка → `run_stage_2()`
3. Референсы лиц → `run_stage_3_refs()`
4. Генерация кадров → `run_stage_4_scenes()`
5. Озвучка → `run_stage_5_yandex_tts()`
6. Субтитры → `wrapper_subtitles()` (гибридный)
7. Манифест → `run_stage_6_manifest()`
8. Рендеринг → `wrapper_render()` (гибридный)

#### Основные методы

**run_pipeline(start_from, custom_idea)**:
- Главный цикл выполнения
- Сохраняет идею в `idea.txt` если передана
- Последовательно выполняет этапы начиная с `start_from`
- Останавливается при первой ошибке (success == False)
- Обновляет статус и логи на каждом шаге
- Finally блок: `is_running = False`

**log(message)**:
- Добавляет timestamp в формате `[HH:MM:SS]`
- Пишет в self.logs и stdout
- Используется для отслеживания прогресса

**load_config()**:
- Читает settings.json
- Возвращает дефолтные значения если файл не найден
- Ключи: use_colab_whisper, use_colab_render, colab_url

#### Proxy управление

**_proxy_off()**:
- Удаляет HTTP_PROXY, HTTPS_PROXY из os.environ
- Используется перед запросами к Colab (ngrok блокирует proxy)

**_proxy_on()**:
- Устанавливает proxy на `http://127.0.0.1:10809`
- Восстанавливается после Colab операций

#### Гибридный режим: Субтитры (wrapper_subtitles)

**Логика выбора**:
```python
use_cloud = config.get("use_colab_whisper", False) and config.get("colab_url", "")
```

**Локальный режим**:
- Вызывает `run_stage_5_5_subtitles()` напрямую

**Colab режим**:
1. Сканирует `outputs/audio/` для всех эпизодов
2. Для каждого эпизода:
   - Получает список аудиофайлов через `_get_episode_audio_files()`
   - Сортирует по scene_id (regex `\d+` из имени файла)
   - Отправляет каждый файл на `/whisper` endpoint
3. Обрабатывает ответ:
   - Извлекает words из `{"0": [words]}` или `{"words": [...]}`
   - Чистит текст: удаляет не-буквенно-цифровые символы, uppercase
   - Округляет тайминги до 3 знаков
4. Сохраняет в `outputs/subtitles/{ep_name}.json`
   - Формат: `{scene_id: [{text, start, end}]}`

**Обработка дублей**:
- Отслеживает seen scene_id
- Пропускает дубликаты с предупреждением

#### Гибридный режим: Рендер (wrapper_render)

**Логика выбора**:
```python
use_cloud = config.get("use_colab_render", False) and config.get("colab_url", "")
```

**Локальный режим**:
- Вызывает `run_stage_7_render()` напрямую

**Colab режим**:
1. Читает `data/3_assembly_manifest.json`
2. Для каждого эпизода в манифесте:
   - Открывает все audio/image файлы сцен
   - Читает субтитры из `outputs/subtitles/{ep_name}.json`
   - Формирует multipart/form-data запрос:
     - `manifest_json`: JSON массив сцен
     - `subtitles_json`: JSON субтитров
     - `settings_json`: конфигурация (quality, aspect_ratio, codec)
     - `audio_files`: массив MP3 файлов
     - `image_files`: массив JPG файлов
3. POST на `/render` endpoint (timeout 900s)
4. Стримит ответ (video/mp4) в `outputs/final_videos/{ep_name}.mp4`
5. Логирует размер полученного файла

**Управление файлами**:
- Использует контекстные менеджеры для file handles
- Finally блок закрывает все открытые файлы
- Предупреждение если субтитры не найдены (рендерит без них)

#### Утилиты

**_get_episode_audio_files(ep_name)**:
- Сканирует `outputs/audio/{ep_name}/`
- Фильтрует .mp3/.wav файлы
- Сортирует по scene_id (regex extraction)
- Возвращает: `[(scene_id, full_path)]`
- Обрабатывает дубли и файлы без номеров

**_colab_url(config)**:
- Извлекает и очищает colab_url (rstrip "/")
- Используется для построения endpoint URLs
