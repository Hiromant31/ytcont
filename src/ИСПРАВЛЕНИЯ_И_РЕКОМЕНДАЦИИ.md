# 🔧 Исправления и Рекомендации для AI Video Studio

## ✅ Исправленные Ошибки

### 1. **SyntaxError в main.py (строка 1041)**
**Проблема:** JavaScript код внутри Python f-string не был правильно заэкранирован.
```python
# БЫЛО (ошибка):
async function openEpisodesModal() {
  const modal = document.getElementById('episodes-modal');
  
# СТАЛО (исправлено):
async function openEpisodesModal() {{
  const modal = document.getElementById('episodes-modal');
```
**Решение:** Все одинарные фигурные скобки `{` и `}` в JavaScript коде внутри f-string заменены на двойные `{{` и `}}`.

---

### 2. **KeyError в main.py (строка 1302)**
**Проблема:** Обращение к несуществующему ключу `response_data["stage_1"]`.
```python
# БЫЛО (ошибка):
if not response_data["stage_1"] and not response_data["stage_2"]:

# СТАЛО (исправлено):
if not response_data["master_story"] and not response_data["stage_2"]:
```
**Решение:** Заменено на правильное поле `master_story`, которое реально создается.

---

### 3. **ValueError в stage_1_story.py (строка 618)**
**Проблема:** Использование пустой переменной `episodes` вместо заполненной `episodes_final`.
```python
# БЫЛО (ошибка):
full_text = "\n\n".join(episodes.values())

# СТАЛО (исправлено):
full_text = "\n\n".join(episodes_final.values())
```
**Решение:** Переменная `episodes` была удалена из кода, используется `episodes_final`.

## 🎯 Рекомендации по Оптимизации Контекста

### Проблема: "Контекст не влезает периодически"

#### 1. **Сократите промпты в stage_1_story.py**

**Текущие промпты очень длинные.** Сократите их:

```python
# ТЕКУЩИЙ ПРОМПТ (слишком длинный):
STORY_PROMPT = """
Ты — сценарист сериалов короткого формата.

На основе идеи создай цельную историю из 3 эпизодов.

Требования:
- 2–3 главных героя
- один центральный конфликт
...
"""

# РЕКОМЕНДУЕМЫЙ ПРОМПТ (короткий):
STORY_PROMPT = """Создай структуру истории из 3 эпизодов.
Формат: TITLE, CHARACTERS, CORE CONFLICT, EP1/EP2/EP3."""
```

#### 2. **Уменьшите max_tokens в API вызовах**

В `stage_1_story.py` и других местах используются большие значения:
```python
# БЫЛО:
max_tokens=8000  # Слишком много!
max_tokens=6000

# РЕКОМЕНДУЕТСЯ:
max_tokens=4000  # Для master story
max_tokens=2000  # Для эпизодов
max_tokens=2000  # Для polish
max_tokens=3000  # Для экстрактора
```

#### 3. **Разбейте длинные промпты на части**

Вместо одного длинного промпта с множеством правил, используйте короткий базовый промпт + повторный вызов для исправлений:

```python
# Первый вызов - генерация
response = call_ai_text(client, ..., simple_prompt, max_tokens=2000)

# Второй вызов - только если нужна доработка
if needs_polish:
    response = call_ai_text(client, ..., polish_prompt, max_tokens=1000)
```

#### 4. **Включите потоковую передачу (streaming)**

Добавьте параметр `stream=True` в API вызовах для экономии памяти:
```python
response = client.chat.completions.create(
    model=model,
    messages=[...],
    stream=True,  # ← Добавьте это
    max_tokens=2000
)
```

#### 5. **Кэшируйте промежуточные результаты**

Сохраняйте результаты каждого этапа и не перегенерируйте их:
```python
# В начале каждой функции проверяйте кэш:
if os.path.exists("data/1_base_structure.json"):
    print("⚡ Используем закэшированный результат Stage 1")
    return True  # Пропускаем генерацию
```

#### 6. **Уменьшите количество ретраев**

В `call_ai_extractor()` слишком много попыток:
```python
# БЫЛО:
MAX_RETRIES = 5  # Слишком много!

# РЕКОМЕНДУЕТСЯ:
MAX_RETRIES = 2  # Достаточно
```

#### 7. **Упростите JSON схему экстрактора**

Текущая схема требует много токенов. Упростите:
```python
# БЫЛО (сложная схема):
{
  "characters": [
    {"name": "...", "role": "...", "appearance": "...", "style": "..."}
  ],
  "setting": {...},
  "visual_style": {...}
}

# РЕКОМЕНДУЕТСЯ (простая схема):
{
  "characters": ["Анна, детектив", "Борис, подозреваемый"],
  "setting": "Современный город, зима",
  "style": "Нуар, тёмные тона"
}
```
