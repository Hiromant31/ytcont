## 🔧 Технические детали

### OAuth 2.0 Flow для YouTube
1. Пользователь создает проект в Google Cloud Console
2. Включает YouTube Data API v3
3. Создает OAuth 2.0 credentials (Desktop App)
4. Скачивает client_secret.json
5. Загружает JSON в веб-интерфейс

### Отложенная публикация
```python
# Использовать scheduleTime параметр
body = {
    "snippet": {
        "title": "Video Title",
        "description": "Description with hashtags",
        "tags": ["tag1", "tag2"],
        "categoryId": "22"  # Entertainment
    },
    "status": {
        "privacyStatus": "private",
        "publishAt": "2024-01-16T10:00:00.000Z",  # scheduleTime
        "selfDeclaredMadeForKids": False
    }
}
```

### Резервный вариант (если scheduleTime не работает)
1. Загрузить как private сразу
2. Сохранить videoId
3. Запланировать videos.update() через cronjob в нужное время

## ⚙️ Рабочий процесс

1. **Настройка:**
   - Пользователь загружает OAuth JSON
   - Настраивает параметры (автозагрузка, генерация тегов)

2. **Во время пайплайна:**
   - После успешного рендеринга (этап 8)
   - Запускается этап 9 (YouTube Upload)
   - Генерируются метаданные из сценария
   - Видео загружается на YouTube

3. **Ручной режим:**
   - Кнопка в UI "Загрузить на YouTube"
   - Выбирает видео из outputs/final_videos/
   - Позволяет редактировать метаданные
   - Загружает сразу или отложенно

## 🛡️ Безопасность

1. **Копирование файлов** - все изменения в копиях
2. **OAuth токены** - хранить локально, не коммитить
3. **Проверка файлов** - валидация перед заменой оригиналов
4. **Откат** - возможность вернуть оригиналы в любой момент
